# Security Policy

## Supported Versions

The following versions of IncidentFlow MCP currently receive security updates.

We recommend always using the latest release.

---

## Reporting a Vulnerability

If you discover a security vulnerability in IncidentFlow MCP, please report it responsibly.

**Do not open a public GitHub issue for security vulnerabilities.**

Instead, please contact the maintainers privately:

Email: incidentflowio@gmail.com

Include the following information:

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if available)

We will acknowledge your report within **72 hours** and work with you to resolve the issue.

---

## Disclosure Policy

Once the vulnerability is confirmed and fixed:

1. A security patch will be released
2. A security advisory will be published
3. Credit will be given to the reporter (if desired)

---

## Scope

This security policy applies to:

- IncidentFlow MCP server
- CLI tools in this repository
- Official Docker images

It does **not** apply to third-party integrations or external services.

---

## Responsible Disclosure

We appreciate responsible disclosure and will work with security researchers to ensure a safe ecosystem for all users.
